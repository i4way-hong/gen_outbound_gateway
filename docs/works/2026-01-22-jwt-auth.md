# 2026-01-22 JWT 인증 전환

## 목표
- Basic Auth 대신 JWT 기반 인증/인가 적용
- 토큰 발급/갱신 API 제공

## 변경 사항
- JWT 토큰 발급/검증 로직 추가 (Access/Refresh)
- `/auth/login`, `/auth/refresh` 엔드포인트 추가
- Spring Security를 JWT Bearer 방식으로 전환
- Swagger 보안 스키마 등록
- 환경 변수 문서화 및 테스트 프로파일 JWT 설정 추가
- DB 사용자 인증(UserDetailsService) 추가 및 `app_users` 엔티티/리포지토리 구성
- 인증 활성화 플래그(`AUTH_ENABLED`) 추가

## 확인
- `mvn -q test` 통과 (AuthControllerTests 포함)

## 메모
- 운영 환경에서는 JWT 시크릿(32바이트 이상)을 반드시 설정
- Refresh 토큰 폐기는 필요 시 Redis/DB로 확장
